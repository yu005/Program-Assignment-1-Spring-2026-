# Running Instance（演算法執行實例）與 Demo

## Demo 名稱
主題樂園最佳遊園計畫 Demo

## 目的
將主題樂園地圖轉成多重權重圖 G=(V,E)，並用 DFS + Branch-and-Bound 找出在限制內可行且最佳的遊園路線。

## 使用方式
1. 用瀏覽器開啟 `index.html`
2. 選擇案例 1 / 2 / 3（或手動輸入限制條件）
3. 按下「▶ 執行 Demo」
4. 觀察右側的 Running Instance 紀錄與中間的最佳路線

---

## 案例 1：70 分鐘限制
- **輸入**：時間 70、預算 300、體力 20、曝曬不限
- **搜尋方法**：DFS 展開所有候選路線
- **剪枝方式**：Branch-and-Bound 排除超限與無法勝出的分支
- **最佳路線**：V1 → V2 → V4 → V2 → V1
- **遊玩設施**：V2 雲霄飛車、V4 鬼屋（2 項）
- **結果**：總時間 70 分鐘、花費 270 元、體力 14、偏好 17
- **說明**：V2 在路線中出現兩次，回程經過不代表重複遊玩

## 案例 2：90 分鐘限制 + 曝曬限制
- **輸入**：時間 90、預算 300、體力 20、曝曬 20
- **最佳路線**：系統依據放寬的時間與曝曬限制搜尋
- **遊玩設施**：3 項（限制放寬後可多玩 1 個設施）
- **說明**：曝曬限制會排除高曝曬路線，最佳解可能不同於不限曝曬的版本

## 案例 3：120 分鐘寬鬆限制
- **輸入**：時間 120、預算 450、體力 25、曝曬 25
- **遊玩設施**：4~5 項（限制大幅放寬）
- **說明**：時間與預算充裕時，系統可找到遊玩更多設施的路線

## 搜尋過程重點

### DFS 展開邏輯
1. 從 V1 出發，沿著實際存在的邊一步一步搜尋
2. 到達未遊玩設施時，分成「遊玩」與「只經過」兩種分支
3. 每當路線回到 V1，檢查是否為新的最佳可行解

### Branch-and-Bound 剪枝
1. **限制剪枝**：時間/預算/體力/曝曬超過限制 → 立即停止
2. **界限剪枝**：樂觀上界的設施數/偏好無法超越目前最佳解 → 停止
3. **狀態支配**：相同位置與選擇但資源消耗更差的狀態 → 捨棄

### 適合報告時講的重點
- 這不是隨機答案，而是逐步搜尋出來的結果
- V2 在路線中出現兩次，是因為回程經過，不代表重複遊玩
- 若限制放寬，系統會自動改選其他路線
- 右側紀錄可以看到每次剪枝的原因（超限/上界/支配）

## 演算法對照
| 演算法 | 概念 | 評估 |
|--------|------|------|
| DFS | 完整展開狀態樹 | 核心搜尋策略 |
| Branch-and-Bound | 提早剪枝不可能的分支 | 搭配 DFS 使用 |
| Best-First Search | 以評估值決定展開順序 | 可優先走高分路徑 |
| A* Algorithm | 加入估計剩餘成本 | 多維限制較難設計啟發函數 |
| Greedy Algorithm | 每步選最佳 | 速度快但不保證最佳 |
| Dynamic Programming | 記憶已算狀態 | 適合重複狀態多的情況 |
